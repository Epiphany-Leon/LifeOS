## 更新记录 1：Xcode 构建 CodeSign 失败根因定位与永久修复

### 目标与问题

- Xcode 每次 Archive / Build 都报 `CodeSign error: code signing failed`，原因是 `Docs/` 目录内的 Markdown 文件携带了 `com.apple.quarantine` 扩展属性（xattr），导致代码签名校验器拒绝对这些文件签名。
- 每次手动执行 `xattr -cr` 只能临时解决，重新拉取或新建文件后问题复发。

### 关键实现

1. **添加 Pre-Archive Build Phase（Shell Script）**  
   在 `NexaLife.xcodeproj/project.pbxproj` 中添加 `PBXShellScriptBuildPhase`：
   
   ```bash
   xattr -cr "${TARGET_BUILD_DIR}/${WRAPPER_NAME}"
   ```
   
   在每次构建前自动清除 bundle 内所有扩展属性。

2. **关闭 User Script Sandboxing**  
   同步设置 Debug 与 Release 配置均为 `ENABLE_USER_SCRIPT_SANDBOXING = NO`，使 Shell Script Build Phase 有权限执行 `xattr` 命令。

3. **将 `Docs/` 目录移出 app bundle**  
   将 `NexaLife/Docs/` 整体迁移到项目根目录 `Docs/`，从 Xcode target membership 中移除。app bundle 体积从 ~30 MB 压缩至 ~1.9 MB。

### 结果

- 后续构建均通过 CodeSign 校验，不再需要手动清除 xattr。

## 更新记录 4：ProjectManagementSheet 完全重写，修复项目创建 Bug

### 目标与问题

- `+` 创建项目后，列表不刷新（`@Query` 不在同一 tick 内更新 `@State` 绑定选中项），导致"创建成功但选中行消失"的 bug。
- 项目缺少 status、startDate、endDate 字段。

### 关键实现

   用 `@State var projects: [ExecutionProject]` 手动维护列表，`reloadProjects()` 通过 `FetchDescriptor` 直接查询数据库，绕过 SwiftData 的 reactive 延迟。

2. **skipNextSelectionReset 哨兵**  
   新建后立即设置 `skipNextSelectionReset = true`，防止 `onChange(of: projects)` 清空选中项。

3. **refreshTick**  
   创建成功后递增 `refreshTick`，强制 SwiftUI 重置表单视图 identity，避免旧草稿残留。

4. **ProjectDraft 值类型**  
   所有可编辑字段集中在 `ProjectDraft` struct，与持久化模型分离。

5. **ExecutionProject 模型扩展**（`ExecutionProject.swift`）  
   添加 `ProjectStatus: String, Codable, CaseIterable`（.notStarted / .inProgress / .finished）；添加 `status`、`startDate`、`endDate` 字段（lightweight migration）。

6. **Markdown 渲染**  
   description 字段通过 `AttributedString(markdown:)` 渲染。

7. **OSLog 诊断**  
   通过 `projectSheetLog` 在关键操作节点记录日志。

### 结果

- 项目创建后立即出现在列表并自动选中，不再丢失。

## 更新记录 5：AI 存储架构重构（Keychain → 本地文件）

### 目标与问题

- API Token 存储在 Keychain，沙盒调试环境下权限问题频繁，且用户无法直观查看 / 迁移 token 文件。
- 需要将存储统一到本地文件，并做一次性 Keychain → 文件迁移。

### 关键实现（`AICredentialStore.swift`）

1. `mode` 属性固定返回 `.localFile`，不再暴露 storage mode 选择器。
2. `bootstrapSecurityDefaults()`：启动时自动将 Keychain 中残留 token 迁移到本地文件，并清除 Keychain 条目。
3. `saveAPIKey(_:)`：直接写入本地文件，同时清除 Keychain（防残留）。
4. `readAPIKey()`：仅从本地文件读取。
5. 文件路径优先级：自定义路径 → `storageDirectory` UserDefaults → ApplicationSupport/NexaLife/ai_api_token.txt。
6. 移除 `saveAPIKey(_:to:)`、`readAPIKey(from:)`、`updateStorageMode()` 等多模式接口。

### 关联变更（`PreferencesView.swift`）

- 移除 Storage Mode picker 和 Local File path 行。
- 移除 `selectTokenFile()` 函数与 Keychain 相关的位置描述逻辑。

### 结果

- 存储路径清晰可见（Settings → AI 面板显示文件路径），不再有 Keychain 权限干扰。

## 更新记录 6：AIService 增强与连接测试 Bug 修复

### 目标与问题

- `runAITest()` 始终显示"连接失败"，实际原因是 `max_tokens: 20` 对 DeepSeek v4-pro 等思考模型来说不足（推理 token 消耗完后内容为空），而非 API 密钥或模型名问题。

### 关键实现（`AIService.swift`）

1. 新增 `callAPIDetailed(prompt:maxTokens:) async -> Result<String, AIServiceError>`：
   - HTTP 200 一律视为成功，返回内容预览。
   - 非 200 时捕获响应 body 片段拼入错误信息。
2. `callAPI()` 改为 `callAPIDetailed` 的薄包装。
3. 新增 `AIServiceError` 枚举：`.notConfigured`、`.http(status, body)`、`.transport`。

### 关联变更（`PreferencesView.swift`）

- `runAITest()` 改调 `callAPIDetailed`，`maxTokens` 提升至 512。
- 测试结果显示实际错误文字或"连接成功：[内容预览]"。

### 结果

- 用户确认"连接成功了"；DeepSeek v4-flash / v4-pro / deepseek-chat / deepseek-reasoner 均可正常测试。

## 更新记录 11：AI 模块分类修复 — classifyText 健壮性与关键词补全

### 目标与问题

- Quick Capture 输入"今天 ChatGPT 会员扣费 20 美元"后，AI 建议始终显示"Inbox"而非"Lifestyle"。
- 两个根本原因：①`classifyText()` 使用 `maxTokens: 20`，思考模型输出被截断或附加额外文字导致 `AppModule(rawValue:)` 精确匹配失败；②离线规则引擎缺少"扣费"、"会员"、"订阅"等关键词，本地兜底也返回 Inbox。

### 关键实现（`AIService.swift`）

1. **`classifyText()` maxTokens 提升**：20 → 60，避免模型输出被截断。
2. **响应解析健壮化**：改为扫描式匹配 —— 用 `localizedCaseInsensitiveContains` 遍历所有 `AppModule` rawValue，第一个匹配的模块即为结果，不再要求精确等值。
3. **离线关键词扩充**：`TaskSuggestionEngine.moduleRules` Lifestyle 条目新增 15 个关键词：
   `扣费 / 充值 / 订阅 / 会员 / 续费 / 购买 / 付费 / 费用 / fee / subscription / membership / purchase / bought / paid / charged`

### 结果

- "今天 ChatGPT 会员扣费 20 美元"无论 AI 是否配置，均正确建议归入 Lifestyle。

  - 右半部分（以 `Divider` 分隔）：`chevron.up.chevron.down` Menu 按钮，列出所有现有项目（含"收件箱（无项目）"选项）。
  - 整体使用 `controlBackgroundColor` 背景 + 8pt 圆角 + 0.2 opacity 描边，视觉上形成统一的输入控件。
- AI Wand 按钮（`wand.and.stars`）保持在控件右侧，点击强制触发 `suggestTaskMetadata()`，自动填入分类、标签与项目名。
- `existingProjectNames` 已合并任务的 `projectName` 与 `ExecutionProject` 列表，下拉选项实时反映数据库状态。

### 结果

- 项目字段兼具"输入新项目"和"下拉选择现有项目"两种交互，AI 自动建议直接写入字段，操作路径缩短。
